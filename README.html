WELCOME TO TEAMS

Teams is a command line based app that allows users to join teams of their peers remotely, set their status, see the stati of their peers, and switch teams.

Some Assumptions:

  • Clients know the IP address and port of the server
  • Clients know the ID of the team they would like to join
  • Clients coordinate amongst themselves to determine which teams they will join
  • Clients are only allowed in 1 team at a time

Directions

Server Admin

To initialize the server, use a command prompt to navigate to the src folder of the app. Once there, use the following command to start the server: java TeamServer. The server admin will be prompted to provide the port number on which the server will listen for incoming messages. After that, the server is initialized and will operate autonomously!

Clients

To get started, use a command prompt to navigate to the src folder of the app. Once there, use the following command to start the app client: java TeamClient. Clients will be prompted to provide the IP address of the server, then the server port number. After that, clients will be able to select from a set of commands:

  • CONN (client name) (team ID) (initial status)
  • UPDATE (client ID) (new status)
  • GET (client ID)
  • SWITCH (client ID) (new team ID)

CONN takes 3 parameters: the client's name in plain text, the ID of the team the client wishes to join, and the client's initial status at the time of connection to the server. Upon using this command, the client will receive an acknowledgement from the server that contains their unique ID. Clients must remember this ID as they will need it when using the other commands.

UPDATE takes 2 parameters: the client's ID (provided after running the CONN command), and the client's new status. This command will cause the server to update it's backend to reflect the client's updated status. Others in the same team will be able to see this new status when they run the GET command.

GET takes 1 parameter: the client's ID (provided after running the CONN command). This command will request from the server the status vector of the client's team, printing a neatly formatted readout with their teammates' names and stati.

SWITCH takes 2 parameters: the client's ID (provided after running the CONN command), and the ID of the team the client would like to join. Running this command changes the client's team from their current team to the team that matches the ID they provide. They are removed from their previous team.
